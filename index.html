AngularJS

HTML for Web Apps

What is Angular JS?

AngularJS is a framework created to assist in creating applications within a browser (can it do more?). The goal of the creators was to make development and testing easier. Using two-way binding, the view and model are kept in synchronization. The framework is easy enough. A developer (perhaps the smart, front-end guy) first marks up an html document with angular tags that correspond to the data that requires dynamicy. The data for the page is provided in the form of a JSON object. Simply stated, the values within the object update the angular tags within the html, making the page dynamic. For example, suppose you want to write a page that provides a list of projects. You wish to use links on the side that, when clicked, filter the results of the product list. All you would need to do is send off an ajax request that returns a json object with information about the desired products and the html elements will automatically update without the need of jQuery.

The Benefits

So why would you ever want to use AngularJS? There are several benefits to using this framework. Some of them are more obvious than others, but the technology is new enough that new ways of using Angular and additional benefits are being discovered all the time. The following are some benefits of adopting this framework.

  • Traditional HTML is extended to better serve dynamic content
  • Lighter web application
  • DOM manipulation is eliminated from the list of things to do
  • Users can bookmark and email links to locations within apps though it uses AJAX by deep linking
  • Angular is not demanding, you can use as much or as little as you'd like

The Basics

ng-app

ng-app is used to identify the angular portion of a page. This allows you to limit the scope to a particular portion.

<html ng-app>

In this case, ng-app is activating the entire page.

ng-modal

ng-modal is used to link the model and the view. If the model is updated via the textbox or input that ng-model is on, the view is updated. The model may also be a variable within a JSON object.

Model: <input type="text" ng-model="yourName" placeholder="Enter a name here"> View: <h1>Hello {{yourName}}!</h1>

In this case, if you type your name in the input box, the text surrounded by curly braces will update with the input box's value.

ng-controller

ng-controller is used to identify the angular portion of a page. This allows you to limit the scope to a particular portion.

  • <div ng-controller="TodoCtrl">
  • <span>{{remaining()}} of {{todos.length}} remaining</span>
  • [ <a href="" ng-click="archive()">archive</a> ]
  • <ul class="unstyled">
  • <li ng-repeat="todo in todos">
  • <input type="checkbox" ng-model="todo.done">
  • <span class="done-{{todo.done}}">{{todo.text}}</span>
  • </li>
  • </ul>
  • <form ng-submit="addTodo()">
  • <input type="text" ng-model="todoText" size="30" placeholder="add new todo here">
  • <input class="btn-primary" type="submit" value="add">
  • </form>
  • </div>

In this case, ng-app is activating the entire page.

Directives

What are directives? All of the ng- prefixed names are directives. Directives are a way of grouping HTML together in order to perform some sort of DOM manipulation. When the complier finds the directive, the function(s) can do just about anything. You can use a directive that already exists, on the Angular website, or create your own to fit the needs of your webpage.

Additionally, a directive extends HTML. Rather than using a span or div, you can then use a more semantic tag such as 'shopping-cart'. In fact, this makes it easy to break HTML into components. Components are useful when you use a portion of HTML over and over again.

For example, suppose I am writing a blog site with multiple articles, sections, and teasers. Instead of writing each document separately, I could write components such as heading, image with caption snippets, and other repeated HTML. The navigation could be a simple component that is included on every page.

Here are some popular directives:

  • ng-app Designates the root element of the angular application.
  • ng-bind Changes the HTML text to the value of a given expression.
  • ng-model Allows two-way binding. If the text on the DOM changes, the variable within the scope changes. If the variable within the scope changes, the text on the DOM changes.
  • ng-class Dynamically loads class attributes.
  • ng-controller Designates a javascript controller class that contains functions and attributes for DOM manipulation
  • ng-repeat Can be used to repeat an HTML element for every value in a javascript object.
  • ng-show Shows a particular element if an expression evaluates to true.
  • ng-hide Shows a particular element if an expression evaluates to true.
  • ng-switch Switches between different HTML template elements according to a selected expression.
  • ng-if Creates or deletes an element according to an expression's evaluation of true or false.

Each directive has a specific purpose. If you're already writing in angular, it's worth researching directives for common jquery functions such as autocomplete, mouseovers, and input.
